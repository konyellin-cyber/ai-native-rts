# Phase 4 — 可扩展架构 Checklist

## Phase 4：可扩展架构

**目标**：建立扩展框架，让新内容可以低成本加入。

**阶段定位**：Phase 4 是"能长"。

### 子阶段 4A：科技树系统

- [ ] **4A.1** 科技树数据结构设计
- [ ] **4A.2** 科技研究 UI
- [ ] **4A.3** 至少 2 个科技效果实现（如：工人速度提升、战士攻击力提升）

### 子阶段 4B：多兵种扩展

- [ ] **4B.1** 兵种定义数据驱动化（从 JSON/Resource 配置）
- [ ] **4B.2** 新兵种实现（如：弓箭手/法师/骑兵）
- [ ] **4B.3** 反制关系（兵种克制链）

### 子阶段 4C：种族系统

- [ ] **4C.1** 种族数据隔离框架
- [ ] **4C.2** 第 2 种族实现（至少 1 个差异化单位）
- [ ] **4C.3** 种族选择 UI

### 子阶段 4D：地形系统

- [ ] **4D.1** 地形类型（平原/森林/山地/水域）
- [ ] **4D.2** 地形影响（移动速度、视野、通行限制）
- [ ] **4D.3** 地图编辑器（简易）
