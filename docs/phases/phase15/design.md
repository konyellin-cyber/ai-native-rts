# Phase 15 设计文档 — 将领单位 + 兵团跟随

**所属项目**: AI Native RTS
**状态**: 草案
**创建**: 2026-04-03
**上游文档**: [gameplay-vision.md](../../design/game/gameplay-vision.md)

## 目标

验证核心手感：**将领能带着人墙在战场上行动，脱离军队独自出击时有紧张感。**

不涉及士气、溃败传染、敌方 AI 三阶段——那些在 Phase 16 验证。

---

## 子阶段拆分

```
Phase 15A — 将领单位
  交付物：一个能在地图上流畅移动的将领单位
  验证点：移动手感是否轻盈即时；阵亡后行为是否正确

Phase 15B — 兵团跟随
  交付物：将领带着密集人墙移动；切换待命后将领可单独行动
  验证点：跟随/待命切换是否自然；人墙是否有密集感而非小队感
```

## Phase 15A：将领单位

### 将领节点设计

将领是独立的 3D 单位节点，与 Fighter / Archer 平级，但行为逻辑不同。

```
GeneralUnit（节点）
├── 视觉：与士兵有明显区分（尺寸更大、颜色/标识不同）
├── 移动：玩家直接控制，即时响应，无惯性延迟
├── HP：有血量，可以被攻击和阵亡
├── 阵营：team_name（red / blue），与士兵一致
└── 标识：unit_type = "general"
```

### 移动手感目标

- 输入 → 移动的响应延迟 < 1帧，无加速度过渡
- 移动速度略快于普通士兵（将领需要能快速穿越战场）
- PC 模式：鼠标右键点击地面 → 将领移动到目标点（与现有右键移动逻辑复用）

### 阵亡逻辑

- 将领 HP 归零 → 播放阵亡动画 → 节点移除
- 阵亡事件发出信号 `general_died(team_name)`，供后续 Phase 的士气系统监听
- MVP 阶段：将领阵亡后游戏不立即结束，但所部失去锚点（15B 实现）

### 与现有系统的关系

| 现有系统 | 处理方式 |
|---------|---------|
| base_unit.gd | 将领继承 base_unit，复用受击白闪、击退、伤害处理 |
| SelectionManager | 将领可被框选/点选，行为与普通单位一致 |
| team_name / unit_type | 沿用现有阵营标识约定 |

## Phase 15B：兵团跟随

### 核心概念：锚点聚集

士兵不被分配进"小队容器"，而是**主动向将领周围密集聚集**。

```
每个士兵的行为优先级（简化）：
1. 如果将领发出"跟随"指令 → 向将领锚点位置靠拢，保持密集队形
2. 如果将领发出"待命"指令 → 停在原位，不跟随
3. 如果将领阵亡 → 锚点消失，进入自由状态（为 Phase 16 士气溃败做准备）
```

### 密集聚集算法

士兵在将领周围的**环形区域**内密集排列，目标是"肩并肩"而非均匀间距。

```
聚集区域：以将领为圆心，半径 R 内
排列方式：士兵目标位置 = 将领位置 + 偏移量（按编号均匀分布在圆内）
密集感来源：士兵间距设为单位碰撞半径的 1.1 倍（略大于碰撞，不重叠但紧靠）
边缘形状：圆形或椭圆形，边缘自然不规则（偏移量加小量随机扰动）
```

### 跟随 / 待命切换

两种状态由玩家主动切换：

| 状态 | 触发方式 | 士兵行为 |
|------|---------|---------|
| 跟随（默认） | 按键 / 手柄键 | 持续向将领锚点靠拢，将领移动时跟随 |
| 原地待命 | 再次按键 | 士兵停在当前位置，不跟随将领移动 |

**待命时将领离开的处理**：
- 将领离开视野范围内 → 士兵阵型开始缓慢松散（间距逐渐扩大）
- 将领回到视野范围内 → 士兵重新向将领靠拢（但不自动切换回跟随状态）

### 哑兵（15A 过渡用）

15A 完成时，场景中放置一批固定数量的 dummy 士兵（无 AI、无战斗），只实现锚点跟随逻辑。用于验证将领手感时有基本语境（"我在带着一支军队"的感受），15B 完成后替换为真实士兵。

## 测试分层约定（策略 B）

从 Phase 15 起，测试目录按生命周期分层：

```
tests/
├── core/       ← 底层行为，跨 Phase 长期有效
│               （伤害计算、死亡信号、碰撞、弹道）
├── legacy/     ← 旧机制测试，对应机制被替代时同步删除
│               （HQ 生产面板、旧 AI 行为等）
└── gameplay/   ← 新玩法测试（将领、兵团、士气、战策时刻）
```

**判断标准**：
- 验证的行为是否与具体玩法机制绑定？→ `gameplay/`
- 验证的是底层引擎行为（物理、信号、渲染）？→ `core/`
- 是否对应一个"已知会被替代"的旧机制？→ `legacy/`

**清理时机**：每个 Phase 末尾，检查 `legacy/` 中是否有对应机制已被新系统覆盖，确认后删除。

**旧测试迁移**：Phase 15 开始时，现有测试按以下规则迁移：

| 旧测试 | 迁移目标 | 理由 |
|-------|---------|------|
| 战斗伤害 / 死亡 / 弹道 | `core/` | 底层行为，新玩法不改变 |
| 框选 / 点选 | `core/` | 输入管线，长期有效 |
| HQ 生产面板 | `legacy/` | 征兵系统上线后废弃 |
| 旧 AI 行为（Fighter 自动攻击等） | `legacy/` 或 `core/`（视情况） | 兵团逻辑上线后部分替代 |

## 验证标准

**15A 完成标准**：
- 将领在地图上移动流畅，响应即时无延迟
- 将领被攻击后 HP 正确扣减，阵亡后节点移除并发出 `general_died` 信号
- headless 测试：将领移动、受击、阵亡三个场景各 PASS

**15B 完成标准**：
- 将领带队移动时，士兵保持密集人墙跟随，目视无明显小队感
- 切换待命后将领独自移动，士兵停留原位
- 将领长时间离开后，原地士兵阵型可见松散
- headless 测试：跟随/待命切换、锚点距离、队形密度各 PASS

### 视觉验证场景设计（general_visual）

主场景（`main.tscn`）混合了工人、HQ、经济 UI、AI 对手和自动化剧本，
不适合目视观察将领/兵团行为。因此独立出专项演示场景：

```
tests/gameplay/general_visual/
├── config.json       仅含将领 + 哑兵配置，无 HQ / 工人 / 矿点
├── bootstrap.gd      继承 GameplayBootstrap；加鼠标右键射线；无断言；无超时
└── scene.tscn        window_mode=true，headless 回归自动跳过
```

**操作手册**：
- 右键点击地面 → 将领带队移动
- Space 键 → 切换跟随/待命（头顶标签变色）
- 待命后右键让将领独走，观察士兵留守
- 再按 Space → 切回跟随，观察士兵重新靠拢
- 手动关窗口退出

**场景内容**：1 名红方将领 + 30 个哑兵，等距相机俯视，无 UI 噪音
