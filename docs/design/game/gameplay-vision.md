# 玩法愿景设计文档

**项目**: AI Native RTS
**状态**: 草案（待细化）
**创建**: 2026-04-03

---

## 核心命题

> 侦查、试探、拉扯、集结——在长时间的博弈张力之后，海量兵力在战场上正面碰撞，一决胜负。

游戏的核心快感分两层：

**前期的张力**：双方都在暗中集结，用小股兵力试探对方底牌。侦骑、断粮、诈退，每一次小规模接触都在积累信息和优势。玩家用战策时刻推演，用信号调度友军，逐渐形成战略判断。

**决战的爆发**：当双方兵力都集结到极限，一场避无可避的大决战爆发。战线正面碰撞，士气溃败像瘟疫蔓延，每个决策的后果在这一刻集中兑现。这场决战的重量，来自前期所有拉扯的积累。

所有机制设计都为这个节奏服务：前期给予足够的信息博弈空间，决战给予足够的规模冲击感。

## 玩家定位：战场上的将领

玩家不是上帝视角的指挥官，而是**战场上的一位将领**。

- 玩家能集结和指挥一支军队（所部）
- 玩家直接在战场上，能感受战线的压力和崩溃的威胁
- 玩家能向友军将领发出信号（增援、侧翼、撤退集结等），但无法直接控制友军
- 友军将领由 AI 驱动，有自己的判断，会响应玩家信号，也会主动行动

这个设定天然适合**手柄操作**：操控一个将领单位 + 发信号，而不是框选和点击。

### 将领与军队的关系

玩家操控将领本人，手感轻盈敏捷（而非拖着整支军队的重量感）。

**将领可以脱离军队独自行动**，并对军队下达两种指令：
- **跟随**：军队以将领为锚点持续跟随移动
- **原地待命**：军队停在原位，将领独自行动

这让将领的每次独自出动都是一个有风险的决策：
- 孤身侦查敌方集结位置
- 冲去友军快崩的战线稳住士气
- 短暂脱离主力去骚扰敌方将领

**代价**：将领阵亡 → 所部立刻失去锚点，士气崩溃传染开始；将领离开视野 → 原地待命的部队阵型逐渐松散。

## 战场规模：人海与阵型

古代战场的核心视觉：**海量士兵、密集人墙、战线推进与崩溃**。

### 兵力规模目标
- 单场战役：双方各数百至千余兵力
- 玩家所部：一支中等规模的军队（数十至百余）
- 友军将领：各自统领数十至百余
- 敌方：由多个 AI 将领分别统帅，整体上与我方相当或更多

### 军队呈现方式：密集阵型 + 个体行为

**每个士兵都是独立个体**（独立 HP、独立士气），但在行动上以将领为中心密集聚集，而不是散兵游勇。

核心视觉原则：士兵肩并肩紧靠，阵型边缘自然不规则，**远看是一堵人墙，不是几个小队方块**。

```
❌ 全面战争式（小队感）：  ● ● ● ● ●   ← 均匀间距，一眼数清
                           ● ● ● ● ●

✅ 本游戏（人墙感）：      ●●●●●●●●   ← 密集紧靠，视觉连成片
                          ●●●●●●●●●
                           ●●●●●●●●
```

**将领是军队的锚点**：
- 士兵主动向将领周围密集聚集，将领移动时军队跟随
- 将领在场 → 士兵保持紧密队形，阵型有纪律
- 将领离开或阵亡 → 士兵失去锚点，队形开始松散

**溃败的视觉**：
- 士兵不是瞬间消失，而是从阵型边缘开始逃散
- 相邻士兵受传染后跟着逃，人墙从边缘开始"融化"
- 将领冲入快溃散的位置 → 逃兵重新向将领靠拢，人墙重新成形

## 士气与溃败传染（核心机制）

单位不仅仅是 HP 归零死亡。**士气崩溃才是战场真正的终结方式。**

### 溃败传染
- 一支部队受到重创 → 士气下降 → 触发**溃逃**（单位开始后退或逃散）
- 溃逃单位影响相邻友军士气 → 引发连锁溃败
- 这就是"濒临崩溃"的来源：一个缺口撕开，恐慌像瘟疫一样蔓延

### 将领稳定效果
- 玩家将领在场 → 周围士兵士气稳定，抵抗溃败传染
- 玩家冲入快要崩溃的战线 → 稳住局势，制造逆转时刻
- 玩家将领阵亡或离开 → 所部士气迅速下滑

### 逆转感的设计
- 敌军海浪般一波接一波涌来
- 玩家用手头的兵力消耗、侧翼、信号友军增援，艰难维持
- 在"再多一波就要崩了"的边缘拿下胜利 → 这就是目标体验

## 敌方 AI 行为：集结、试探、决战

游戏支持随意暂停，所以敌方不能是塔防式的"无脑一波波送"——那种设计只在实时压力下有意义。

**敌方 AI 应该像一支真正的军队在行动**，有战略节奏：

### 三个阶段

**1. 集结期**
- 敌方将领各自在后方或占领据点周围征兵，军队规模缓慢增长
- 玩家能在地图上观察到敌方兵力在某处聚集，有时间做出预判
- 此阶段敌方不主动进攻，但会争夺中立据点

**2. 试探期**
- 敌方派出小股兵力骚扰、试探玩家防线
- 目的：侦查玩家兵力分布、消耗玩家弹药（士气）、寻找薄弱点
- 玩家若应对过度（倾巢而出去追一支小队），则暴露主力位置
- 玩家若置之不理，试探部队会蚕食边缘据点

**3. 决战期**
- 敌方集结达到一定规模 + 找到战机 → 发动总攻
- 多路并进，大规模兵团正面对撞
- 这才是"海量兵力大战一波"的时刻，也是最容易触发溃败传染的时刻

### 为什么这个节奏更好

| 塔防节奏 | 本设计节奏 |
|---------|-----------|
| 被动等波次，无聊 | 主动侦查/应对，有张力 |
| 每波压力均匀 | 压力从低到高，决战时爆发 |
| 暂停没有意义（只是减速） | 暂停用于判断集结动向、制定预案 |
| 敌方无记忆 | 敌方根据试探结果调整主攻方向 |

### 暂停的新意义 → 战策时刻（详见下节）
- 暂停不是减速，而是进入一种特殊的指挥状态
- 战场视角拉远，局势一览无余
- 可以为自己和友军下达计策，恢复运行后生效

## 经济系统：领地与补给

古代战争不采矿。经济基础是**领土控制和人口动员**。

### 领地控制
- 地图上分布着村庄、城镇、关隘等据点
- 控制据点 → 定期获得兵力补充（征兵/民兵加入）
- 失去据点 → 补充来源断绝，兵力只出不进

### 补给线
- 从己方后方基地延伸到前线
- 补给线被截断 → 前线部队士气惩罚、不能补充
- 保护补给线本身构成战术目标

### 不设资源采集
- 没有矿工、采矿点、资源数字
- "经济"体现在占领据点的速度和战线稳定性上
- 拖延时间本身就是风险（敌方也在征兵）

## 信号系统：与友军将领协作

玩家不能直接控制友军，但可以发出**信号**影响他们的决策。

### 信号类型（草案）
| 信号 | 含义 | 友军响应 |
|------|------|---------|
| 增援 | 我这里快撑不住 | 最近的友军将领向玩家位置移动 |
| 侧翼 | 绕到敌军侧面 | 接收者尝试迂回包抄 |
| 集结 | 收缩集中兵力 | 所有友军将领向指定点集结 |
| 进攻 | 全线压上 | 友军主动出击而非防守 |
| 撤退 | 放弃当前位置 | 友军向后撤至下一个据点 |

### AI 将领的自主性
- 友军 AI 会优先响应玩家信号，但也有自己的判断
- 信号是"建议"，不是命令——AI 将领可能因局势判断而调整
- 这让每场战役都有不可预测性，结果不是完全由玩家脚本决定

## 战策时刻：暂停不是暂停，是指挥权

游戏支持随时暂停，但"暂停"本身被包装成一个有重量的玩法动作，而不是简单的时间静止。

### 核心概念：战策时刻（Battle Pause）

触发暂停时：
- 战场**视角拉远**，从将领视角切换到鸟瞰全图
- 时间静止，但玩家进入一个专属的**指挥界面**
- 可以为**自己的部队**和**每位友军将领**分别下达一条计策
- 恢复运行后，所有计策在接下来的若干秒内生效

### 计策类型（草案）

**战术计策**（影响部队行动）
| 计策 | 效果 |
|------|------|
| 急袭 | 部队冲刺移速+50%，持续10秒，士气消耗加快 |
| 坚守 | 原地防御，受击伤害-30%，不主动追击 |
| 穿插 | 部队绕至目标侧翼，忽视正面接触 |
| 断粮 | 派小队袭击敌方补给线，而非正面交战 |

**士气计策**（影响士气状态）
| 计策 | 效果 |
|------|------|
| 鼓舞 | 目标将领所部士气+20，缓解溃败传染 |
| 诈退 | 假装溃败诱敌深入，下一次反冲击伤害+40% |
| 摇旗 | 提振全军士气，范围效果，冷却时间长 |

**情报计策**（影响侦查和信息）
| 计策 | 效果 |
|------|------|
| 侦骑 | 侦查指定区域敌方兵力和集结情况 |
| 疑兵 | 在某处制造虚假兵力信号，迷惑敌方AI |

### 可升级的指挥能力

战策时刻是**可成长的技能**，而非固定功能：

| 等级 | 解锁内容 |
|------|---------|
| Lv.1（初始） | 只能给自己下1条计策；视野拉远有限 |
| Lv.2 | 可以同时给1位友军将领下计策 |
| Lv.3 | 可同时给所有友军下计策；计策效果持续时间+50% |
| Lv.4 | 解锁高阶计策（诈退、疑兵等）；视野可全图查看集结动向 |

升级来源：战役中控制关键据点（军师台、鼓楼等），或在战役积分中解锁。

### 设计意图

- 暂停变成"值得等待的资源"，玩家会选择在关键时机使用
- 给随意暂停的操作加上**策略深度**，而不只是缓解操作压力
- 可升级意味着玩家有成长感，高等级指挥能力带来质变体验
- 与手柄操作兼容：进入战策时刻后用摇杆/方向键选计策，不需要精确鼠标

## 可重复性：随机沙盒 + Roguelike 元素

游戏不是线性关卡，而是**每次不同的沙盒战役**。

### 随机化元素
- 地形：随机生成山地、平原、河流、关隘
- 派系：随机对手（不同战术风格的 AI 派系）
- 初始兵力配置：玩家和友军的兵种组合略有不同
- 关键据点位置：影响战役节奏的核心地点随机分布

### Roguelike 元素
- 每场战役中，玩家将领可获得**战场特质**（临时加成）
- 例如："今日士气如虹"（全军士气+20%）、"奇袭专家"（侧翼攻击伤害+30%）
- 战役结束后，积累永久成长（将领声望、解锁新战法）

### 沙盒感
- 没有"通关条件"，每场战役在一定条件下结束（一方溃败或主城陷落）
- 玩家可以选择进攻节奏，快速决战或消耗拖延
- 局势可逆：早期劣势可以通过信号协调和士气逆转翻盘

## 手柄操作设计

操控一位将领比框选单位对手柄更自然。

### 核心操作
- **左摇杆**：移动将领
- **右摇杆**：调整所部行军方向/阵型朝向
- **扳机键**：冲锋/突击命令
- **方向键**：发信号（↑增援 ↓集结 ←侧翼 →进攻）
- **B/Circle**：撤退/重组
- **A/Cross + 目标选择**：指定进攻目标

### 感受目标
- 在战场上移动、感受战线压力
- 冲入快要崩溃的地方稳住局势，有英雄感
- 发信号协调全局，不需要精确的鼠标点击

## 与现有系统的关系

| 现有系统 | 保留/改造/废弃 |
|---------|--------------|
| Fighter / Archer 兵种 | 保留，作为步兵/远程基础单位 |
| 阵营 Red vs Blue | 保留框架，扩展为多将领结构 |
| 生产系统（HQ 面板） | 改造为"征兵"（消耗据点控制权，而非资源） |
| SelectionManager 框选 | 保留用于 PC 模式；手柄模式走将领直控 |
| SimulatedPlayer / 测试体系 | 完整保留，新机制同样需要自动化测试 |

## 核心玩法 MVP 规划

验证目标：**侦查拉扯 → 大决战这个节奏，玩起来真的有快感吗？**

### 必须有（验证核心快感）

**将领控制**
- 将领是独立单位，手感轻盈敏捷，移动即时响应
- 两种军队指令：**跟随**（军队持续跟随将领）/ **原地待命**（将领独自行动）
- 将领阵亡 → 所部立刻失去锚点，溃败传染开始

**兵团跟随行为**
- 士兵以将领为锚点密集聚集，肩并肩形成人墙
- 将领带队移动时，军队保持密集队形跟随
- 将领命令原地待命后，部队停留原位；将领长时间离开则队形逐渐松散

**士气 + 溃败传染**
- 受重创 → 士气下降 → 单位溃逃（从阵型边缘开始）
- 溃逃传染相邻单位，人墙从边缘"融化"
- 将领冲入溃散区域 → 逃兵重新向将领靠拢，人墙重新成形

**敌方 AI 三阶段**
- 集结期：在后方积累兵力，可被侦查
- 试探期：派小股骚扰，探查玩家位置
- 决战期：集结到位 + 找到战机 → 全线压上

**规模感**
- 双方同屏兵力各 100–200 可流畅运行

### 简化处理

| 完整愿景 | MVP 简化版 |
|---------|-----------|
| 多位友军将领 + 信号系统 | 1位友军将领，只响应"增援"信号 |
| 战策时刻可升级（Lv.1–4） | 只有 Lv.1：暂停 + 给自己下1条计策（急袭/坚守） |
| 领地经济 + 征兵 | 固定时间间隔自动补兵：双方将领周期性获得新士兵加入，无需玩家操作；敌方 AI 越到后期补兵越快（模拟集结期结束） |
| 随机地形 + Roguelike | 固定地图，少量据点但不影响兵力 |
| 手柄支持 | 先 PC 键盘鼠标，手柄后续 |

### 验证标准

MVP 跑完后，用这三个问题判断方向是否正确：

1. **将领手感**：将领移动是否流畅自然？脱离军队侦查时有没有紧张感？
2. **张力感**：决战前有没有"山雨欲来"的紧迫感，会不会主动去侦查？
3. **决战冲击**：两军碰撞时有没有规模感？士气溃败传染看得出来吗？

### 建议 Phase 拆分

```
Phase 15 — 将领单位 + 兵团跟随 + 士气原型
  验证点：将领手感是否对；跟随/待命切换是否自然；
          溃败传染是否有视觉感；将领冲入溃散区能否稳住

Phase 16 — 敌方 AI 三阶段 + 决战感
  验证点：前期试探有没有信息博弈感；
          决战兵力规模是否有冲击感
```
