## 层级结构

| 层 | 节点类型 | 职责 |
|----|---------|------|
| 游戏世界 | Node3D | 地图、单位、HQ、矿物 |
| 面板层 | Control（全屏） | ProdPanel 等浮动面板 |
| HUD 层 | CanvasLayer | BottomBar（始终置顶） |

## 设计原则

- **极简**：不追求 StarCraft 级 UI 复杂度
- **底部状态栏固定**，游戏区域铺满视口
- **逻辑与渲染分离**：每个 UI 组件分为数据逻辑层（信号、状态）和视觉渲染层（Control 节点），headless 下只跳过渲染层
- **原生 Control 节点**：不依赖外部主题系统
- **面板悬浮在对象上方**：生产面板等跟随 3D 对象屏幕投影坐标定位

## 摄像机规格（Phase 9 等距视角）

| 参数 | 值 | 说明 |
|------|----|------|
| projection | ORTHOGONAL | 正交投影，无透视形变 |
| rotation_degrees | (-45, -45, 0) | 俯仰 -45°，偏航 -45°，经典等距方向 |
| size | 2000 | 正交视口垂直世界单位 |
| cam_height | 1500 | 摄像机 Y 高度 |
| lateral | cam_height / √2 ≈ 1061 | 视线水平偏移量（居中修正值） |
| position | (map_w/2 - lateral, 1500, map_h/2 + lateral) | ≈ (219, 1500, 1893) for 2560×1664 |

**居中原理**：等距 45° 偏航摄像机的视线方向 forward=(0.5, -0.707, -0.5)，
视线打到 y=0 平面的 X、Z 偏移均为 `cam_height/√2`，
因此需将摄像机向左前方偏移 lateral 才能使视线交点落在地图中心。

**断言验证（camera_isometric）**：
- `rotation_degrees.y ≈ -45° ±5°`
- 视线射到 y=0 的交点与地图中心偏差 ≤ 300 单位
